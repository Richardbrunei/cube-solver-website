What is this?

Rubik's Cube Interactive bridges the gap between your physical cube and the digital world. Whether you're a speedcuber analyzing patterns, a beginner learning algorithms, or just someone who loves the iconic puzzle, this tool lets you capture, visualize, and manipulate cube states with ease.

Built with vanilla JavaScript and powered by Python computer vision, it combines the simplicity of web technologies with the accuracy of HSV-based color detection to bring your cube to life on screen.

Features

  • 3D Visualization: Interactive 3D cube with CSS transforms and smooth animations. Rotate and explore from any angle.
  • Net View: Flattened layout showing all six faces simultaneously, perfect for pattern recognition and editing.
  • Smart Camera Capture: Advanced HSV color detection using OpenCV automatically identifies cube colors, even in challenging lighting.
  • Manual Color Editing: Fine-tune any sticker with a click. Choose from standard cube colors (White, Yellow, Red, Orange, Blue, Green).
  • Instant Reset: Return to solved state with one click, complete with confirmation to prevent accidents.
  • Responsive Design: Optimized for desktop, tablet, and mobile. Your cube looks great everywhere.

How to Use

  1. View Your Cube: Toggle between 3D and Net views to see your cube from different perspectives.
  2. Capture with Camera: Click the Camera button and follow the guided sequence (White → Red → Green → Yellow → Orange → Blue) to scan your physical cube.
  3. Edit Manually: Click any sticker to change its color. Perfect for corrections or creating custom patterns.
  4. Reset Anytime: Hit the Reset button to return to the classic solved state.

Technology

This project showcases modern web development without the complexity of build tools or frameworks:

  • Frontend: Vanilla JavaScript (ES6 modules), CSS3 (Grid, Flexbox, 3D transforms), HTML5 Web APIs
  • Backend: Python 3.7+ with Flask, OpenCV for computer vision, NumPy for color processing
  • Architecture: Modular ES6 design with centralized state management and observer pattern
  • No Build Process: Direct browser execution, keeping things simple and fast

The Cube State Format

Under the hood, cube states are represented as a 54-character "cubestring" following standard notation:

UUUUUUUUURRRRRRRRRFFFFFFFFFDDDDDDDDDLLLLLLLLLBBBBBBBBB

This represents the six faces in order: Up (White), Right (Red), Front (Green), Down (Yellow), Left (Orange), Back (Blue). Each face has 9 stickers, reading left-to-right, top-to-bottom.
